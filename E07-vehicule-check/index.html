Exercice 7 - car-check

Objectifs de la leçon

  • Comprendre les éléments principaux de l'analyse à l'aide d'un exemple

Durée estimée : 3h

Type d'enseignement : exercice, apprentissage personnel, support des autres étudiants et du professeur.

Enoncé

Une entreprise a de nombreux véhicules, principalement utilitaires, mis à disposition pour différents travaux. Elle décide de développer une application pour gérer la location (interne) de ces véhicules et en particulier la vérification du véhicule après utilisation.

Le projet visual paradigm réalisé pour ce projet est disponible ici, car-check.vpp

Avertissement : il s'agit ici d'un document en construction.

Analyse fonctionnelle

Business process

Le processus de location d'un véhicule est le suivant :

Les différents acteurs sont :
  • worker : le travailleur qui a besoin d'un véhicule
  • fleet manager : le responsable de la flotte de véhicule
both signifie les deux.
Les étapes du processus sont :
  1. le worker demande un véhicule
  2. le fleet manager vérifie la demande
  3. le fleet manager accepte ou refuse la demande
  4. le fleet manager sélectionne un véhicule
  5. le worker et le fleet manager vérifie le véhicule
  6. le travailleur utilise le véhicule
  7. ensemble, ils vérifient le véhicule au retour (check out)
  8. si des problèmes sont détectés, le fleet manager rédige un rapport
