Cours Analyse Informatique

Leçon T00 - Analyse Technique

Objectifs de la leçon

Comprendre l'analyse technique

Après cette leçon, l'étudiant saura

  • Ce qu'est l'analyse technique
  • Quelles sont les différentes vues de l'analyse technique

Durée estimée : 1 heure.

Type d'enseignement : ex-cathédra & démo & discussions.

Introduction

Pour rappel, l'analyse est décomposée en deux phases:

  • l'analyse fonctionnelle qui répond à la question de ce qu'il faut faire
  • l'analyse technique qui répond à la question de comment il faut le faire

L'analyse technique vient donc de manière naturelle après l'analyse fonctionnelle, car comment définir le comment réaliser quelque chose si nous ne connaissons pas le ce que nous devons réaliser.

En pratique, les deux analyses sont bien souvent menées de front, avec une petite avance pour l'analyse fonctionnelle, et de nombreux aller-retour entre les deux phases sont effectués.

L'analyse technique comprend de nombreuses vues, dépendant de vos besoins, vous utiliserez l'une ou l'autre de ces vues:

  1. class model est le modèle de classe, il documente les classes d'implémentation et il utilise les diagrammes UML de classes, d'objets, de séquences, de collaboration, d'activités et d'états.
  2. compoment model est le modèle des composants, il documente les composants utilisés dans l'application et il utilise le diagramme UML de composants.
  3. deployment model est le modèle de déploiement, il documente comment les composants sont déployés et utilise le diagramme UML de déploiement.
  4. database model est le modèle de base de données, il documente la structure de la base de données et il utilise des diagrammes ERD ou relationnel.
  5. navigation model est le modèle de navigation, il documente la navigation d'écrans en écrans et il utilise un simple diagramme de lien entre écrans, par exemple le diagramme UML d'activités.
  6. UI model est le modèle d'interface utilisateur, il documente les interfaces utilisatures, donc, en général, les écrans, et il comprendra essentiellement des wireframes, des maquettes.
  7. interface model est le modèle d'interface, il documente les autres interfaces, les interfaces avec d'autres ordinateurs ou avec des équipements électroniques, il utilisera entre autres des diagrammes UML de séquences.
